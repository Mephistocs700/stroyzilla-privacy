Политика конфиденциальности StroyZilla

Дата вступления в силу: 17 июня 2026 г.

Настоящая Политика конфиденциальности описывает, какие данные обрабатывает приложение StroyZilla, как они используются, где хранятся и какие права есть у пользователя.

1. Общая информация

StroyZilla — приложение для подрядчиков, прорабов, мастеров и частных специалистов, которое помогает вести строительные и ремонтные объекты, учитывать расходы и доходы, создавать сметы, прикреплять фотографии, формировать отчёты и синхронизировать данные между устройствами.

Разработчик приложения: Andrei Gursky / Gursky Works.

Контакт для вопросов по конфиденциальности: cs70001@gmail.com

2. Какие данные может обрабатывать приложение

Приложение может обрабатывать следующие данные, которые пользователь добавляет самостоятельно:

  • данные строительных и ремонтных объектов;
  • названия объектов, адреса или описания объектов, если пользователь указывает их вручную;
  • расходы, доходы, суммы, категории, комментарии и даты операций;
  • сметы, позиции смет, цены, объёмы работ и материалов;
  • фотографии объектов, этапов работ, материалов, чеков или документов;
  • PDF-отчёты и документы, созданные или загруженные пользователем;
  • данные резервных копий и синхронизации;
  • данные аккаунта Google, если пользователь использует вход через Google.

Приложение не требует от пользователя вводить паспортные данные, банковские реквизиты, данные банковских карт или другую платёжную информацию.

3. Как используются данные

Данные используются для следующих целей:

  • создание и ведение строительных или ремонтных объектов;
  • учёт расходов, доходов и финансового результата по объектам;
  • создание смет и отчётов;
  • прикрепление фотографий к объектам и этапам работ;
  • формирование PDF-документов;
  • резервное копирование и синхронизация данных между устройствами пользователя;
  • обеспечение работы входа через Google;
  • улучшение стабильности и работы приложения.

4. Вход через Google

Приложение может использовать вход через Google для идентификации пользователя и синхронизации данных между устройствами.

При использовании входа через Google приложение может получить базовую информацию аккаунта, например идентификатор пользователя, имя, адрес электронной почты и изображение профиля, если эти данные доступны через Google Sign-In.

Эти данные используются только для входа в приложение, привязки данных к аккаунту пользователя и синхронизации.

5. Хранение данных

Данные могут храниться:

  • локально на устройстве пользователя;
  • в облачных сервисах Firebase / Google Cloud, если пользователь использует функции синхронизации, резервного копирования, хранения фотографий или входа через Google.

Для хранения и синхронизации данных приложение может использовать Firebase Authentication, Firebase Realtime Database и Firebase Storage.

6. Передача данных третьим лицам

Разработчик не продаёт пользовательские данные третьим лицам.

Данные могут обрабатываться сторонними сервисами, необходимыми для работы приложения, включая сервисы Google Firebase / Google Cloud. Эти сервисы используются для авторизации, хранения данных, хранения файлов, синхронизации и обеспечения стабильной работы приложения.

Данные также могут быть раскрыты, если это требуется по закону, решению суда или законному запросу государственных органов.

7. Фотографии и документы

Пользователь самостоятельно выбирает, какие фотографии, документы и сведения добавлять в приложение.

Перед загрузкой фотографий или документов пользователь должен убедиться, что имеет право хранить и обрабатывать такие материалы, особенно если на них присутствуют персональные данные, изображения людей, адреса, документы заказчиков, чеки или иная конфиденциальная информация.

8. Безопасность данных

Приложение использует стандартные технические средства защиты, включая защищённую передачу данных через сервисы Google и Firebase.

Несмотря на принимаемые меры, ни один способ хранения или передачи данных через интернет не может быть абсолютно безопасным. Пользователь несёт ответственность за безопасность своего устройства, аккаунта Google и доступа к приложению.

9. Удаление данных

Пользователь может удалить данные, созданные в приложении, через функции приложения, если такие функции доступны.

Для запроса на удаление аккаунта, облачных данных, фотографий или резервных копий пользователь может написать на адрес: cs70001@gmail.com

В запросе необходимо указать адрес электронной почты Google-аккаунта, который использовался в приложении. После получения запроса разработчик удалит доступные данные пользователя в разумный срок, если их хранение не требуется по закону или техническим причинам резервного копирования.

10. Дети

Приложение не предназначено для детей и не ориентировано на пользователей младше 13 лет.

StroyZilla предназначено для взрослых пользователей, занимающихся строительством, ремонтом, подрядными работами, сметами и учётом объектов.

11. Изменения политики

Разработчик может периодически обновлять настоящую Политику конфиденциальности. Новая версия политики публикуется по той же ссылке, которая указана в Google Play и внутри приложения.
